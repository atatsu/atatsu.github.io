Module giblets.widgets.progressbar

Progressbar widget.

This progressbar is very similar to the one you get with awful.widget.progressbar, except that it is always drawn with segments, or ticks. In addition, rather than being drawn with a solid border and ticks inside the border, each tick is drawn as though it is its own entity. So if a border is used, all four sides of each tick will have a border.

All the functions implemented by awful.widget.progressbar are also implemented here. So a giblets.widgets.progressbar can be a drop-in replacement for any awful.widget.progressbars that are in use. Keep in mind though that giblets.widgets.progressbar does have additional functions so the reverse isn't true.

Progressbar:set_tooltip ([enabled=true[, fmt_func]])
Enable or disable the progressbar tooltip (not compatible with awful.widget.progressbar).

Signals:

    widget::updated

Parameters:

  • enabled bool Enable or disable the tooltip. (default true)
  • fmt_func func An optional function that receives the current progressbar value and should return a string to be used as the tooltip text. The default format function is: function(value) return value * 100 .. "%" end. (optional)

opts
Progressbar options. If an option is available via your beautiful theme it is listed immediately following the field description. Take note that the theme overrides default options, but any options supplied here will override those found in the theme.

Fields:

  • width int Width of the progressbar.

    theme.giblets.widgets.progressbar.width (default 100)

  • height int Height of the progressbar.

    theme.giblets.widgets.progressbar.height (default 12)

  • vertical bool Draw the progressbar vertically. (default false)
  • max_value number Maximum value the progressbar should support. (default 1)
  • foreground_color string Foreground color of the progressbar. Or in other words, the filled color of ticks.

    theme.giblets.widgets.progressbar.foreground_color (default "#8585ac")

  • background_color string Background color of the progressbar. Or in other words, the empty color of ticks.

    theme.giblets.widgets.progressbar.background_color (default "#484874")

  • border_color string Border color (if border_width is non-zero) of the progressbar. Again, affects the border surrounding each tick.

    theme.giblets.widgets.progressbar.border_color (default "#8585ac")

  • border_width int Pixel width of the border surrounding each tick.

    theme.giblets.widgets.progressbar.border_width (default 1)

  • ticks_gap int Spacing between each tick.

    theme.giblets.widgets.progressbar.ticks_gap (default 3)

  • ticks_size int or table Size of each tick. Can be a number representing both width and height or a table with separate width and height dimentions with the keys width and height ({width = 10, height = 5}).

    theme.giblets.widgets.progressbar.ticks_size (default 8)

  • ticks_align string Set the alignment of the ticks. Can be one of center, top, bottom.

    theme.giblets.widgets.progressbar.ticks_align (default "center")

  • tooltip bool If true a tooltip will be displayed when the progressbar is hovered over displaying the current % value.

    theme.giblets.widgets.progressbar.tooltip (default true)

  • tooltip_fmt_func func An optional function that receives the current progressbar value and should return a string to be used as the tooltip text. The default format function is: function(value) return value * 100 .. "%" end. (optional)
